# 上线灰度与回滚手册

## 0. 发布前检查

- `nginx -t` 校验语法通过。
- Java 包可启动，健康接口可访问。
- Redis / MySQL 连接正常，无认证报错。
- 备份旧版本：
  - `www.xucanwei.xyz.conf.bak.<time>`
  - `picture-backend-0.0.1-SNAPSHOT.jar.bak.<time>`

## 1. 灰度发布（10% -> 100%）

1. CDN 回源权重先放 10% 到新配置源站（或新实例）。
2. 观察 30 分钟，重点看：
   - 4xx、5xx 比例
   - `/api/ws` 握手失败率
   - 429 命中量是否异常高
   - Java 进程 CPU、GC、线程池
3. 指标正常后提升到 50%，再观察 20 分钟。
4. 最后全量 100% 切流。

## 2. 回滚触发条件

出现以下任一情况立即回滚：

- 5xx 比例 > 1% 且持续 5 分钟
- 登录/上传接口大量超时
- WebSocket 握手成功率持续低于 95%
- Redis 或 MySQL 连接异常不可恢复

## 3. 回滚步骤

1. CDN 流量切回旧源站或旧权重。
2. 源站恢复旧 Nginx 配置并 reload。
3. 后端恢复旧 JAR 包并重启。
4. 核验核心路径：
   - 首页加载
   - 登录
   - 图片列表
   - 图片上传
   - WebSocket 编辑

## 4. 发布后 24 小时巡检

- 每 10 分钟检查一次错误率（前 2 小时），之后改为每小时。
- 检查 `SECURITY_ALERT` 日志是否有异常尖峰。
- 检查 CDN 回源流量与源站带宽是否异常增长。
